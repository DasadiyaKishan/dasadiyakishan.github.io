How I work

Four steps,
repeated until it ships.

I don't disappear after launch. Most projects roll into a small monthly retainer for bug fixes, small features, and dependency updates — only if you want it.
01
Scope

One call, then a written plan: features, tech, timeline, and what's explicitly out.

Week 0 · Async
02
Build

Weekly demos on a staging URL. Loom walkthroughs so you don't have to schedule a meeting.

Week 1—N · Iterative
03
Harden

Tests on critical paths, queue + cache where it matters, deploy automation set up once.

Pre-launch · Boring
04
Hand off

README, env keys, runbook, and a 30-day support window. No mystery code.

Launch · Documented
