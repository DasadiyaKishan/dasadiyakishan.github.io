Problem

Public-health data was scattered across feeds and PDFs. People wanted a single page that loaded fast, told the truth, and didn't lie about the freshness of the numbers.

What I built

Scheduled ingest

Cron-driven jobs pull from official sources, normalise into a flat schema, and write a daily snapshot. If a feed misses, the dashboard says so — it doesn't pretend.

Cache-first reads

Every public route is served from Redis with a short TTL. The DB only sees a query when the cache is genuinely cold.

Honest charts

Plain HTML/CSS for the lists, a small chart lib for the trends. Last-updated stamp visible on every page.

Outcome

Held up during traffic spikes when official sites went down. Used as a backup reference by local communities.
